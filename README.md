# Linux集群版本发布系统

## 项目概述

这是一个基于C/S架构的Linux集群版本发布系统，用于向集群中的节点批量推送文件、配置和应用程序，并支持高级的灰度发布策略。

## 系统架构

### 服务端 (Server)
- **核心架构**：基于Django框架开发的Web应用
- **通信协议**：基于HTTP RESTful API
- **数据库**：MySQL
- **功能**：管理客户端节点、存储发布任务和版本信息、提供Web管理界面

### 客户端 (Client)
- **核心功能**：作为守护进程在Linux节点上运行，与服务端保持长连接，执行发布任务
- **通信**：定期发送心跳包，接收并执行服务端指令
- **文件备份**：在每次发布前自动备份旧文件

## 核心功能

### 1. 服务端核心功能
- **用户认证**：基于账号密码的登录系统
- **节点管理**：注册、心跳检测、状态监控
- **分组管理**：创建、编辑、删除节点分组
- **任务管理**：创建、启动、暂停发布任务
- **灰度发布**：支持动态调整灰度比例
- **版本管理**：语义化版本校验，确保只有新版本才会被发布

### 2. 客户端核心功能
- **节点注册**：首次启动时向服务端注册
- **心跳机制**：定期发送心跳包保持连接
- **任务执行**：下载文件、备份旧文件、部署新文件
- **状态上报**：向服务端上报任务执行结果
- **文件备份**：自动备份被覆盖的旧文件

## 技术栈

- **服务端**：Django 4.2, Python 3.13, MySQL
- **客户端**：Python 3.13, requests库
- **前端**：HTML5, CSS3, JavaScript

## 安装与部署

### 服务端部署
1. **安装依赖**
   ```bash
   pip install django mysqlclient
   ```

2. **配置数据库**
   修改 `deploy_system/settings.py` 中的数据库配置：
   ```python
   DATABASES = {
       'default': {
           'ENGINE': 'django.db.backends.mysql',
           'NAME': 'database',
           'USER': 'user',
           'PASSWORD': 'password',
           'HOST': '0.0.0.0',
           'PORT': '3306',
       }
   }
   ```

3. **初始化数据库**
   ```bash
   python manage.py makemigrations
   python manage.py migrate
   ```

4. **创建超级用户**
   ```bash
   python manage.py createsuperuser
   ```

5. **启动服务**
   ```bash
   python manage.py runserver 0.0.0.0:8000
   ```

### 客户端部署
1. **安装依赖**
   ```bash
   pip install requests
   ```

2. **配置客户端**
   将 `deploy_client.py` 复制到目标Linux节点上，并设置可执行权限：
   ```bash
   chmod +x deploy_client.py
   ```

3. **启动客户端**
   ```bash
   python3 deploy_client.py http://服务器IP:8000
   ```

   建议将客户端配置为系统服务，确保其在系统启动时自动运行。

## 使用方法

### 1. 登录系统
访问 `http://服务器IP:8000/login/`，使用超级用户账号登录。

### 2. 管理节点分组
1. 点击左侧导航栏的「分组管理」
2. 点击「添加分组」按钮创建新分组
3. 可以编辑或删除现有分组

### 3. 创建发布任务
1. 点击左侧导航栏的「发布任务」
2. 点击「创建任务」按钮
3. 填写任务信息：
   - 任务名称
   - 版本号（遵循语义化版本规范）
   - 发布内容来源（本地上传或URL下载）
   - 目标目录
   - 目标分组
   - 初始灰度比例
4. 点击「创建」按钮提交任务

### 4. 执行发布任务
1. 在任务列表中找到创建的任务
2. 点击「启动」按钮开始灰度发布
3. 可以点击「暂停」按钮暂停发布
4. 可以调整灰度比例，逐步扩大发布范围

### 5. 监控发布状态
- 在任务列表中查看任务状态
- 可以查看每个节点的执行状态
- 系统会自动记录发布日志

## 系统特点

1. **可靠性**：基于HTTP协议的可靠通信，支持节点自动重连
2. **安全性**：基于账号密码的认证系统
3. **灵活性**：支持本地上传和URL下载两种发布方式
4. **可控性**：支持动态灰度发布，可随时调整发布范围
5. **可追溯性**：完整的版本记录和文件备份机制
6. **可扩展性**：模块化设计，易于添加新功能

## 注意事项

1. **数据库配置**：确保MySQL数据库服务正常运行，且配置的账号具有足够的权限
2. **文件权限**：客户端需要有目标目录的读写权限
3. **网络连接**：确保客户端与服务端之间网络畅通
4. **备份空间**：确保客户端节点有足够的磁盘空间用于文件备份
5. **版本号格式**：遵循语义化版本规范，如 v1.0.0, 2.1.3 等

## 故障排查

### 客户端无法注册
- 检查服务端地址是否正确
- 检查网络连接是否正常
- 检查服务端是否正在运行

### 任务执行失败
- 检查目标目录权限
- 检查文件下载链接是否有效
- 查看客户端日志（/var/log/deploy_client.log）

### 节点状态显示为非活跃
- 检查客户端是否正在运行
- 检查网络连接是否正常
- 检查客户端日志是否有错误信息
